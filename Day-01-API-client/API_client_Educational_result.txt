درس‌ها و نکات مهمی که در این چالش می‌آموزید عبارت‌اند از:

1. درک مفاهیم API و HTTP:
روش درخواست HTTP (HTTP Request Method): ما با روش HTTP POST کار می‌کنیم که برای ارسال داده‌های جدید به سرور استفاده می‌شود.
URL درخواست (Request URL): آشنایی با URLی که به عنوان آدرس مقصد سرور استفاده می‌شود.
بدنه درخواست (Request Body): ارسال داده‌های JSON به سرور که در اینجا مثالی از "payload": "hello world!" داریم.

2. استفاده از Postman:
یاد می‌گیرید چگونه درخواست‌ها را در Postman ایجاد کنیم و درخواست‌هایی که از طریق یک API ارسال می‌شوند را مدیریت کنیم.
یاد می‌گیریم که چگونه درخواست را ذخیره کنیم و تغییرات ایجاد شده را مدیریت کنیم.

3. تجزیه و تحلیل پاسخ‌های سرور:
کد وضعیت HTTP (HTTP Status Code): کدهای پاسخ HTTP که نشان می‌دهد آیا درخواست موفق بوده یا خطایی رخ داده است.
زمان پاسخ (Response Time): اندازه‌گیری مدت زمانی که طول می‌کشد تا سرور به درخواست ما پاسخ دهد.
حجم پاسخ (Response Size): مقدار داده‌هایی که در پاسخ سرور دریافت می‌شود.
بدنه پاسخ (Response Body): بررسی دقیق محتوای JSON که سرور در پاسخ به درخواست ما ارسال کرده است.

4. به‌روزرسانی و تست مجدد:
تغییر مقدار payload و ارسال مجدد درخواست، به شما این امکان را می‌دهد تا تأثیر تغییرات را مشاهده کنید و نحوه عملکرد API را بهتر درک کنید.
